Prompt Control Plane

Deterministic prompt governance for production LLM systems. 19 tools, policy enforcement, tamper-evident audit trail, zero LLM calls inside.

TypeScript MCP Node.js npm GitHub Action Enterprise
Product Site View on GitHub npm GitHub Action

The problem

Models are versioned. Pipelines are monitored. Access is controlled. But prompts, the instructions shaping model behavior, are often unmanaged text passed between teams. Most organizations cannot easily answer: who changed this prompt? What risk does it introduce? Can prior output be reproduced deterministically? At scale, that gap becomes operational risk.

What I built

A deterministic governance layer for the prompt surface. Prompt Control Plane enforces structure on every prompt before it reaches a model: risk scoring, policy enforcement, structured compilation, and a tamper-evident audit trail. It ships as an MCP server (19 tools), a CLI linter, and a GitHub Action for CI/CD integration. Four tiers from free to enterprise. Works with any codebase: Python, Java, Go, Rust, or any language. Node.js is only needed to run the MCP server process itself.

Key capabilities

  • Risk scoring: Scores prompts across hallucination, ambiguity, scope expansion, and constraint conflict dimensions (14 detection rules)
  • Policy enforcement: Advisory or enforce mode. Blocks deployment when policy violations are detected
  • Structured compilation: Compiles prompts for Anthropic, OpenAI, and Google Gemini with deterministic output
  • Model routing: Deterministic cross-provider routing with full decision-path audit trail (11 models across 4 providers)
  • Audit trail: Hash-chained, tamper-evident JSONL logging. If any entry is modified, all subsequent hashes break
  • Config lock: Passphrase-protected configuration locking for governance control
  • Context compression: Smart compression pipeline (H1-H5 heuristics) with zone protection for structured content
  • Custom rules: User-defined detection rules with regex patterns, severity levels, and risk dimensions
  • Session lifecycle: Full export, delete, and purge controls with retention policies

Design principles

Zero LLM calls run inside the engine. All intelligence comes from the host AI client. The engine provides structure, rules, and discipline. This means: fully deterministic (same input, same output, every time), offline-first (no network dependency), zero marginal cost (no token spend), and reproducible (audit-grade consistency). It integrates into existing CI/CD workflows, it does not replace them.

  • Power (₹899/mo): Unlimited optimizations, always-on mode, 60 requests/min
