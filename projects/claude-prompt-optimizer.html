Claude Prompt Optimizer MCP

An MCP server that analyzes, scores, and compiles prompts for maximum impact and minimum cost — zero LLM calls, all deterministic.

TypeScript MCP Node.js Prompt Engineering npm
View on GitHub

The problem

Vague prompts waste tokens, produce weak outputs, and cost more than they should. Most people don't realize their prompt is ambiguous until after Claude has already processed it. There's no pre-flight check — no way to know if your prompt will produce the output you actually wanted before spending the tokens.

What I built

An MCP server with 5 tools that acts as a prompt quality gate before execution. The key constraint: zero LLM calls. All analysis is deterministic — regex, heuristics, and rule engines. This means the optimizer itself costs nothing to run and adds negligible latency.

The 5 tools

  • optimize_prompt — Analyzes raw prompts, detects ambiguities across 10 rules, scores quality 0-100, and estimates token cost
  • refine_prompt — Iterative refinement loop that incorporates answers to blocking questions and re-scores
  • approve_prompt — Sign-off gate that refuses to approve if blocking questions remain unanswered
  • estimate_cost — Standalone token estimator for any text, with model-specific pricing
  • compress_context — Context pruner that strips irrelevant code and docs before submission (57% token savings in testing)

How it works

The optimizer classifies each prompt against 13 task-type patterns — code, writing, research, planning, analysis, communication, data, and more. It auto-detects audience, tone, and platform (across 19 audience patterns and 9 platforms including Slack, LinkedIn, and email), applies task-type-aware goal enrichment, and uses intent-first classification to prevent misclassifying writing prompts as code. It also detects multi-task overload, identifies high-risk domains, and recommends the right model tier (Haiku for simple tasks, Sonnet for moderate, Opus for complex). The compiled output uses XML-tagged Anthropic-optimized formatting — structured for maximum signal extraction by Claude.

Why it matters

Every prompt that goes through this server is clearer, cheaper, and more likely to produce the output you actually wanted. The 57% context compression alone means you can fit substantially more relevant information into each API call. And because the entire system is deterministic, it runs at zero marginal cost — no API calls, no token spend, no latency tax.
