The problem

Hiring doctrine usually lives as oral tradition: each interviewer interprets the bar differently, notes are inconsistent, and calibration decays over time. That means the company has a philosophy, but not an operating system.

What I built

RishiOS encodes the operating logic directly into software. It can:

  • score candidates on a shared framework
  • generate job descriptions aligned to the rubric
  • build executive briefs
  • structure loose interview notes
  • generate calibration references
  • adapt through override patterns without losing the base doctrine

Why it matters

This is probably the cleanest expression of the whole portfolio thesis. It is not just a tool; it is an attempt to make talent judgment legible, repeatable, and improvable across time and across teams.
